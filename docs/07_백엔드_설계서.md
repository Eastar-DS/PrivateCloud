## 1. 개요 및 기술 스택

### 1.1 설계 목적

본 문서는 sesac-smartdocs 프로젝트의 Spring Boot 기반 REST API 백엔드에 대한 상세 설계를 정의한다. 문서 업로드, AI 자동 태깅, 태그 검색, 테넌트 격리 등 핵심 비즈니스 로직을 처리하는 애플리케이션 계층의 구조와 동작 방식을 명세한다.

### 1.2 설계 범위

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        백엔드 설계 범위                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   In-Scope (담당 영역)                                                      │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  - REST API 설계 및 구현 (문서 CRUD, 태그, 검색, 사용자, 감사로그)  │  │
│   │  - DB 스키마 설계 (MariaDB, Flyway 마이그레이션)                    │  │
│   │  - RBAC 기반 접근 제어 (테넌트 격리, 역할별 권한)                   │  │
│   │  - 비동기 문서 태깅 처리 (RabbitMQ Producer/Consumer)               │  │
│   │  - 오브젝트 스토리지 연동 (Ceph RGW S3 API)                        │  │
│   │  - 감사 로그 (AOP 기반 자동 기록)                                   │  │
│   │  - API 명세 자동 생성 (Swagger UI / OpenAPI 3.0)                    │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Out-of-Scope (범위 외)                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  - Frontend UI 구현 (별도 프론트엔드 설계서)                        │  │
│   │  - AI 모델 학습 및 파인튜닝 (AI Worker 영역)                       │  │
│   │  - OCR 엔진 구현 (AI Worker에서 처리)                               │  │
│   │  - 인프라 프로비저닝 (컴퓨트/네트워크/스토리지 설계서 참조)         │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.4 Read/Write 분리

Primary(쓰기) / Replica(읽기) 분리를 `AbstractRoutingDataSource`로 구현한다.

```
┌─────────────────────────────────────────────────────────────┐
│              Read/Write Split (DataSource 라우팅)             │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│   Spring Boot Application                                     │
│   ┌─────────────────────────────────────────────────────┐   │
│   │          AbstractRoutingDataSource                   │   │
│   │   ┌──────────────────┬──────────────────────┐       │   │
│   │   │  @Transactional  │  @Transactional       │       │   │
│   │   │  (readOnly=false)│  (readOnly=true)      │       │   │
│   │   │       │          │         │              │       │   │
│   │   │       ▼          │         ▼              │       │   │
│   │   │  Primary DS      │    Replica DS          │       │   │
│   │   └──────────────────┴──────────────────────┘       │   │
│   └─────────────────────────────────────────────────────┘   │
│               │                          │                    │
│               ▼                          ▼                    │
│       ┌──────────────┐          ┌──────────────┐            │
│       │ MariaDB      │  복제    │ MariaDB      │            │
│       │ Primary      │ ──────> │ Replica      │            │
│       │ (Write)      │         │ (Read)       │            │
│       │ dev-data-    │         │ dev-data-    │            │
│       │ db-pri       │         │ db-rep       │            │
│       └──────────────┘          └──────────────┘            │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**라우팅 규칙:**
- `@Transactional(readOnly = true)`: Replica DataSource로 라우팅
- `@Transactional` 또는 `@Transactional(readOnly = false)`: Primary DataSource로 라우팅
- 기본값: Primary DataSource

## 3. 배포 및 설정

### 3.1 application.yml 구조

```yaml
spring:
  profiles:
    active: dev

  # --- DataSource (Primary / Replica) ---
  datasource:
    primary:
      url: jdbc:mariadb://dev-data-db-pri:3306/smartdocs
      username: ${DB_USERNAME}
      password: ${DB_PASSWORD}
      driver-class-name: org.mariadb.jdbc.Driver
      hikari:
        maximum-pool-size: 20
        minimum-idle: 5
        connection-timeout: 3000
        idle-timeout: 600000
    replica:
      url: jdbc:mariadb://dev-data-db-rep:3306/smartdocs
      username: ${DB_REPLICA_USERNAME}
      password: ${DB_REPLICA_PASSWORD}
      driver-class-name: org.mariadb.jdbc.Driver
      hikari:
        maximum-pool-size: 30
        minimum-idle: 10
        connection-timeout: 3000

  # --- JPA ---
  jpa:
    hibernate:
      ddl-auto: validate
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MariaDBDialect
        format_sql: false
    open-in-view: false

  # --- Flyway ---
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true

  # --- RabbitMQ ---
  rabbitmq:
    host: dev-svc-app-01
    port: 5672
    username: ${RABBITMQ_USERNAME}
    password: ${RABBITMQ_PASSWORD}
    virtual-host: /smartdocs
    listener:
      simple:
        concurrency: 3
        max-concurrency: 10
        retry:
          enabled: true
          max-attempts: 3
          initial-interval: 1000

  # --- Multipart ---
  servlet:
    multipart:
      max-file-size: 100MB
      max-request-size: 100MB

# --- Ceph S3 ---
storage:
  s3:
    endpoint: http://ceph-rgw.vlan60:7480
    region: default
    access-key: ${S3_ACCESS_KEY}
    secret-key: ${S3_SECRET_KEY}
    bucket-prefix: documents

# --- SpringDoc (Swagger) ---
springdoc:
  api-docs:
    path: /v3/api-docs
  swagger-ui:
    path: /swagger-ui
    tags-sorter: alpha
    operations-sorter: method
  default-consumes-media-type: application/json
  default-produces-media-type: application/json

# --- Actuator ---
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  endpoint:
    health:
      show-details: when-authorized
      group:
        readiness:
          include: db,rabbit,s3
        liveness:
          include: ping
  health:
    rabbit:
      enabled: true

# --- Logging ---
logging:
  level:
    root: INFO
    com.sesac.smartdocs: DEBUG
    org.springframework.security: WARN
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"

# --- Server ---
server:
  port: 8080
  shutdown: graceful
  tomcat:
    max-threads: 200
    accept-count: 100
```

### 3.5 로깅 설정

| 항목 | 설정 |
|------|------|
| **포맷** | JSON 구조화 로깅 (Logback JSON Encoder) |
| **로테이션** | 일별 회전, 30일 보관 |
| **출력 경로** | `/var/log/smartdocs/application.log` |
| **감사 로그** | 별도 파일 `/var/log/smartdocs/audit.log` |
| **최대 크기** | 파일당 500MB, 총 10GB |

```xml
<!-- logback-spring.xml (핵심 설정) -->
<appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>/var/log/smartdocs/application.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
        <fileNamePattern>/var/log/smartdocs/application.%d{yyyy-MM-dd}.log</fileNamePattern>
        <maxHistory>30</maxHistory>
        <totalSizeCap>10GB</totalSizeCap>
    </rollingPolicy>
    <encoder class="net.logstash.logback.encoder.LogstashEncoder">
        <includeMdcKeyName>tenantId</includeMdcKeyName>
        <includeMdcKeyName>userId</includeMdcKeyName>
        <includeMdcKeyName>traceId</includeMdcKeyName>
    </encoder>
</appender>
```
