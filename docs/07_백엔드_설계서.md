## 1. 개요 및 기술 스택

### 1.1 설계 목적

본 문서는 본 프로젝트의 Spring Boot 기반 REST API 백엔드에 대한 상세 설계를 정의한다. 문서 업로드(텍스트/파일), AI 자동 태깅(OCR + KoBERT), 태그 검색, 부서별 접근 제어(RBAC) 등 핵심 비즈니스 로직을 처리하는 애플리케이션 계층의 구조와 동작 방식을 명세한다.

### 1.2 설계 범위

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        백엔드 설계 범위                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   In-Scope (담당 영역)                                                      │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  - REST API 설계 및 구현 (문서 업로드/조회, 태그, 검색, Job, 사용자) │  │
│   │  - DB 스키마 설계 (MariaDB)                                        │  │
│   │  - RBAC 기반 접근 제어 (부서별 격리, 역할별 권한)                   │  │
│   │  - 비동기 문서 태깅 처리 (RabbitMQ Producer + Spring AMQP Consumer) │  │
│   │  - 사전 태깅 문서 업로드 (tags 파라미터 분기 처리)                │  │
│   │  - AI 서버 HTTP 연동 (OCR/태깅 FastAPI 호출)                       │  │
│   │  - 오브젝트 스토리지 연동 (Ceph RGW S3 API)                        │  │
│   │  - API 명세 자동 생성 (Swagger UI / OpenAPI 3.0)                    │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Out-of-Scope (범위 외)                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  - Frontend UI 구현 (별도 프론트엔드 설계서)                        │  │
│   │  - AI 모델 학습 및 파인튜닝 (AI 서버 영역)                         │  │
│   │  - OCR 엔진 구현 (AI 서버 OCR FastAPI에서 처리)                    │  │
│   │  - 인프라 프로비저닝 (컴퓨트/네트워크/스토리지 설계서 참조)         │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.1 전체 시스템 구성도

```
사용자
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                   프론트엔드 계층 (VLAN 10 - DMZ)              │

│          :80/443                   :80/443                   │

                           │ Reverse Proxy
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              애플리케이션 계층 (VLAN 20 - 백엔드망)              │
│                                                               │
│   dev-svc-app-01               dev-svc-app-02               │
│   Spring Boot :8080            Spring Boot :8080             │
│   + RabbitMQ :5672             (AMQP Consumer 포함)          │
│   + AMQP Consumer                                            │
│   + Swagger UI :8080/swagger-ui                              │
└───────┬──────────┬──────────────────┬─────────────────────┘
        │          │                  │
        │          │                  │ HTTP (REST)
        │          │                  ▼
        │          │          ┌────────────────────────┐
        │          │          │ AI 서버 (VLAN 30 - AI망) │
        │          │          │ dev-ai-worker-01        │
        │          │          │ OCR FastAPI :8001       │
        │          │          │ 태깅 FastAPI :8000      │
        │          │          └────────────────────────┘
        │          │
        ▼          ▼
   ┌─────────┐  ┌──────────┐
   │MariaDB  │  │Ceph RGW  │
   │Pri/Rep  │  │S3 API    │
   │VLAN 40  │  │VLAN 50   │
   └─────────┘  └──────────┘
```

> **요청 흐름**: 사용자 → nginx (LB) → Spring Boot (REST API) → MariaDB / Ceph S3 / RabbitMQ → AMQP Consumer → HTTP → AI 서버 (OCR/태깅) → DB/S3 저장
>
> **사전 태깅 흐름**: `tags` 파라미터 전달 시 → 문서 저장 + 태그 즉시 저장 (Job 생성/RabbitMQ 발행 생략)

### 2.4 Read/Write 분리

Primary(쓰기) / Replica(읽기) 분리를 `AbstractRoutingDataSource`로 구현한다.

```
┌─────────────────────────────────────────────────────────────┐
│              Read/Write Split (DataSource 라우팅)             │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│   Spring Boot Application                                     │
│   ┌─────────────────────────────────────────────────────┐   │
│   │          AbstractRoutingDataSource                   │   │
│   │   ┌──────────────────┬──────────────────────┐       │   │
│   │   │  @Transactional  │  @Transactional       │       │   │
│   │   │  (readOnly=false)│  (readOnly=true)      │       │   │
│   │   │       │          │         │              │       │   │
│   │   │       ▼          │         ▼              │       │   │
│   │   │  Primary DS      │    Replica DS          │       │   │
│   │   └──────────────────┴──────────────────────┘       │   │
│   └─────────────────────────────────────────────────────┘   │
│               │                          │                    │
│               ▼                          ▼                    │
│       ┌──────────────┐          ┌──────────────┐            │
│       │ MariaDB      │  복제    │ MariaDB      │            │
│       │ Primary      │ ──────> │ Replica      │            │
│       │ (Write)      │         │ (Read)       │            │
│       │ dev-data-    │         │ dev-data-    │            │
│       │ db-pri       │         │ db-rep       │            │
│       └──────────────┘          └──────────────┘            │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**라우팅 규칙:**
- `@Transactional(readOnly = true)`: Replica DataSource로 라우팅
- `@Transactional` 또는 `@Transactional(readOnly = false)`: Primary DataSource로 라우팅
- 기본값: Primary DataSource

## 3. 배포 및 설정

### 3.1 application.yml 구조

```yaml
spring:
  profiles:
    active: dev

  # --- DataSource (Primary / Replica) ---
  datasource:
    primary:
      url: jdbc:mariadb://dev-data-db-pri:3306/sesac_smartdocs
      username: ${DB_USERNAME}
      password: ${DB_PASSWORD}
      driver-class-name: org.mariadb.jdbc.Driver
      hikari:
        maximum-pool-size: 20
        minimum-idle: 5
        connection-timeout: 3000
        idle-timeout: 600000
    replica:
      url: jdbc:mariadb://dev-data-db-rep:3306/sesac_smartdocs
      username: ${DB_REPLICA_USERNAME}
      password: ${DB_REPLICA_PASSWORD}
      driver-class-name: org.mariadb.jdbc.Driver
      hikari:
        maximum-pool-size: 30
        minimum-idle: 10
        connection-timeout: 3000

  # --- JPA ---
  jpa:
    hibernate:
      ddl-auto: validate
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MariaDBDialect
        format_sql: false
    open-in-view: false

  # --- RabbitMQ ---
  rabbitmq:
    host: dev-svc-app-01
    port: 5672
    username: ${RABBITMQ_USERNAME}
    password: ${RABBITMQ_PASSWORD}
    virtual-host: /sesac-smartdocs
    listener:
      simple:
        concurrency: 3
        max-concurrency: 10
        retry:
          enabled: true
          max-attempts: 3
          initial-interval: 1000

  # --- Multipart ---
  servlet:
    multipart:
      max-file-size: 20MB
      max-request-size: 20MB

# --- AI 서버 ---
ai:
  ocr:
    url: http://dev-ai-worker-01:8001
    timeout: 60000
  tagging:
    url: http://dev-ai-worker-01:8000
    timeout: 30000

# --- Ceph S3 ---
storage:
  s3:
    endpoint: http://ceph-rgw.vlan50:7480
    region: default
    access-key: ${S3_ACCESS_KEY}
    secret-key: ${S3_SECRET_KEY}
    bucket-prefix: documents

# --- SpringDoc (Swagger) ---
springdoc:
  api-docs:
    path: /api/v3/api-docs
  swagger-ui:
    path: /api/swagger-ui
    tags-sorter: alpha
    operations-sorter: method
  default-consumes-media-type: application/json
  default-produces-media-type: application/json

# --- Actuator ---
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  endpoint:
    health:
      show-details: when-authorized
      group:
        readiness:
          include: db,rabbit,s3
        liveness:
          include: ping
  health:
    rabbit:
      enabled: true

# --- Logging ---
logging:
  level:
    root: INFO
    com.sesac.smartdocs: DEBUG
    org.springframework.security: WARN
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"

# --- Server ---
server:
  port: 8080
  shutdown: graceful
  tomcat:
    max-threads: 200
    accept-count: 100
```

| **rabbit** | RabbitMQ | Connection / Channel Test | 태깅 큐 불가 알림 |
| **s3** | Ceph RGW | HeadBucket 호출 | 파일 업로드 불가 알림 |

| **최대 크기** | 파일당 500MB, 총 10GB |

```xml
<!-- logback-spring.xml (핵심 설정) -->
<appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>/var/log/sesac-smartdocs/application.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
        <fileNamePattern>/var/log/sesac-smartdocs/application.%d{yyyy-MM-dd}.log</fileNamePattern>
        <maxHistory>30</maxHistory>
        <totalSizeCap>10GB</totalSizeCap>
    </rollingPolicy>
    <encoder class="net.logstash.logback.encoder.LogstashEncoder">
        <includeMdcKeyName>departmentId</includeMdcKeyName>
        <includeMdcKeyName>userId</includeMdcKeyName>
        <includeMdcKeyName>traceId</includeMdcKeyName>
    </encoder>
</appender>
```
