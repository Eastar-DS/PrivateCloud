## 2. 팀별 역할 정의

### 2.1 컴퓨트/개발 팀 (2명)

#### 2.1.1 역할 개요

AI 플랫폼 인프라의 컴퓨트 계층을 담당하며, GPU 가상화 설계와 AI 워크로드 배포를 통해 인프라 동작을 검증한다.
테넌트는 업무 시스템(웹/앱/DB)을 운영하지 않으며, 데이터 업로드용 최소 VM만 선택적으로 제공한다.

#### 2.1.2 담당 영역

| 영역 | 세부 내용 |
|------|----------|
| **리소스 풀 설계** | 클러스터 풀 구성, 오버커밋 정책 (CPU 2:1, Memory 1:1) |
| **VM 배치 전략** | Affinity/Anti-Affinity 규칙, 장애 도메인 분리 |
| **리소스 쿼터** | 테넌트별 API 사용량 제한, 공용 GPU 사용량 쿼터 |
| **고가용성(HA)** | VM 자동 재시작, Live Migration 설정 |
| **GPU 가상화** | vGPU (NVIDIA GRID), MIG, Passthrough 설계 |
| **GPU 할당 정책** | 테넌트별 vGPU 쿼터, 공용 GPU 풀 스케줄링 |
| **VM 템플릿** | AI 학습 VM, AI 추론 VM, 품질 게이트 VM, 데이터 업로드 VM (테넌트 선택) |
| **검증 워크로드** | 민원 자동 분류 AI 모델 배포, 4계층 데이터 흐름 검증 |

#### 2.1.3 GPU 설계 담당 상세

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    컴퓨트/개발 팀 GPU 설계 범위                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1. GPU 가상화 방식 선정                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • vGPU (NVIDIA GRID): 다수 VM에 GPU 공유, 동적 할당              │  │
│   │  • MIG (Multi-Instance GPU): A100/H100 하드웨어 분할              │  │
│   │  • Passthrough: 단일 VM에 전체 GPU 할당 (학습용)                  │  │
│   │  • Time-slicing: 시간 분할 공유                                   │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   2. GPU 할당 정책 수립                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 테넌트별 vGPU 쿼터 (예: 테넌트 A - 8GB vGPU 2개)              │  │
│   │  • 공용 GPU 풀 Fair-Share 스케줄링                                │  │
│   │  • 작업 우선순위 및 큐잉 정책                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   3. AI VM 템플릿                                                          │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • AI 학습 VM: GPU Passthrough, 대용량 메모리                     │  │
│   │  • AI 추론 VM: vGPU 공유, 중간 메모리                             │  │
│   │  • 품질 게이트 VM: CPU only, 데이터 검증/익명화 처리              │  │
│   │  • 데이터 업로드 VM: 테넌트 선택, 최소 사양 (2vCPU/4GB)           │  │
│   │  • 사전 설치: CUDA, cuDNN, PyTorch/TensorFlow                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.1.4 필요 역량

- KVM/QEMU 하이퍼바이저 이해
- 가상화 기술 (VM, 컨테이너) 기초
- Linux 시스템 관리
- **GPU 가상화 기술 이해 (vGPU, MIG, Passthrough)**
- **AI/ML 프레임워크 기초 (PyTorch, TensorFlow)**
- AI 워크로드 배포 및 검증 능력

#### 2.1.5 학습 키워드

```
- KVM 하이퍼바이저
- 리소스 풀 (Resource Pool)
- Affinity / Anti-Affinity
- 장애 도메인 (Fault Domain)
- Live Migration
- VM 템플릿
- Cloud-init
- NVIDIA vGPU / GRID
- MIG (Multi-Instance GPU)
- GPU Passthrough
- CUDA / cuDNN
```

### 2.2 네트워크/보안 팀 (2명)

#### 2.2.1 역할 개요

네트워크 계층의 설계 및 보안 아키텍처를 담당하며, 멀티테넌시 환경에서의 격리, 접근제어, 데이터 품질 게이트를 구현한다.

#### 2.2.2 담당 영역

| 영역 | 세부 내용 |
|------|----------|
| **망분리 설계** | 9개 네트워크 존 정의, VLAN 할당 계획, IP 대역 설계 |
| **마이크로세그멘테이션** | 테넌트 간 격리, 계층 간 통신 정책 |
| **방화벽 정책** | 계층별 접근 제어 규칙 정의 |
| **로드밸런서** | L4/L7 로드밸런싱, URL 기반 라우팅 |
| **RBAC 설계** | 역할 정의 (플랫폼관리자/테넌트관리자/사용자), 권한 매핑 |
| **암호화 체계** | TLS 1.3 설정, 저장 데이터 AES-256 암호화 정책 |
| **감사 로깅** | 접근 로그, 감사 추적, 테넌트별 로그 분리 |
| **데이터 품질 게이트** | 데이터 검증, 익명화 처리, 승인 프로세스 설계 |

#### 2.2.3 데이터 품질 게이트 담당 상세

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                 네트워크/보안 팀 품질 게이트 설계 범위                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   데이터 품질 게이트는 AI 플랫폼의 계층 2에 해당하며,                       │
│   테넌트 데이터가 공용 AI 플랫폼에 반영되기 전 검증을 수행한다.             │
│                                                                             │
│   1. 데이터 검증 정책                                                       │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 스키마 검증: 데이터 형식, 필수 필드 확인                         │  │
│   │  • 품질 점수: 완전성, 정확성, 일관성 평가                          │  │
│   │  • 이상치 탐지: 비정상 데이터 식별                                  │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   2. 익명화 처리                                                            │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • PII 탐지: 개인정보 자동 식별                                    │  │
│   │  • 마스킹: 민감 정보 비식별화                                      │  │
│   │  • k-익명성: 재식별 방지                                           │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   3. 승인 프로세스                                                          │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 자동 승인: 품질 기준 충족 시                                    │  │
│   │  • 수동 승인: 관리자 검토 필요 시                                  │  │
│   │  • 감사 로깅: 데이터 출처 추적, 변환 이력 기록                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.2.4 네트워크 존 설계 (9개 VLAN)

|---------|-----------|------|

| **30** | DB망 | 공용 DB, 메타데이터 |

- **데이터 보안 및 개인정보 보호**
- **감사 로깅 체계**

#### 2.2.6 학습 키워드

### 2.3 스토리지 팀 (2명)

#### 2.3.1 역할 개요

AI 플랫폼 인프라의 스토리지 계층을 담당하며, 테넌트 데이터 저장과 AI 워크로드 스토리지를 통합 설계한다.
테넌트는 업무 DB를 운영하지 않으며, 민원 데이터 업로드 영역만 제공한다.

#### 2.3.2 담당 영역

| 영역 | 세부 내용 |
|------|----------|
| **분산 스토리지 설계** | Ceph 클러스터 구성, 복제 정책 (replica 3) |
| **티어링 정책** | Hot (NVMe) / Warm (SATA SSD) / Cold (HDD) 정의, 자동 이동 규칙 |
| **테넌트 격리** | 테넌트별 Ceph Pool 분리, 쿼터 설정 |
| **AI 학습 스토리지** | 고처리량 Hot Tier, 병렬 읽기 지원 |
| **모델 저장소** | 모델 레지스트리 전용 Pool, 버전 관리 |
| **스냅샷/백업** | 일일/주간 스냅샷 정책, 백업 자동화 |
| **DR 설계** | RPO/RTO 정의, 복제 전략 (설계 문서) |
| **성능 최적화** | 캐시 설정, I/O 튜닝 |

#### 2.3.3 AI 스토리지 설계 상세

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    스토리지 팀 AI 워크로드 설계 범위                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   AI 워크로드는 일반 업무 대비 높은 I/O 성능을 요구한다.                    │
│   스토리지 팀은 다음 영역을 설계한다:                                       │
│                                                                             │
│   1. AI 학습 데이터 스토리지                                                │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • Hot Tier 배치: 학습 데이터는 NVMe SSD에 저장                    │  │
│   │  • 병렬 읽기: 다수 GPU 노드에서 동시 데이터 읽기 지원               │  │
│   │  • 고대역폭: AI 스토리지망 (VLAN 60) 전용 사용                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   2. 모델 저장소 (Model Registry)                                          │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 전용 Ceph Pool: model-registry-pool                             │  │
│   │  • 버전 관리: 모델 버전별 스냅샷                                   │  │
│   │  • 빠른 배포: 추론 서버에서 모델 로드 최적화                        │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   3. 체크포인트 저장                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 학습 중 체크포인트 저장 (Warm Tier)                             │  │
│   │  • 자동 정리: 오래된 체크포인트 Cold Tier로 이동                   │  │
│   │  • 복구 지원: 학습 재개 시 체크포인트 로드                          │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   4. 스토리지 풀 구성                                                       │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  테넌트 전용       │  공용 AI 플랫폼                                │  │
│   │  ─────────────────┼──────────────────────────────────────────────  │  │
│   │  tenant-a-pool    │  ai-training-pool (학습 데이터)                │  │
│   │  tenant-b-pool    │  model-registry-pool (모델 저장)               │  │
│   │  tenant-c-pool    │  checkpoint-pool (체크포인트)                  │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.3.4 필요 역량

- 스토리지 기초 (블록/파일/오브젝트)
- RAID 개념
- 분산 스토리지 이해
- 백업/복구 개념

```
│   ┌─────────────┐         ┌─────────────┐         ┌─────────────┐          │

│          │                       │                       │                  │
```

### 4.2 협업 상세

| 협업 영역 | 관련 팀 | 협업 내용 |
|----------|--------|----------|
| **VM-네트워크 연결** | 컴퓨트 + 네트워크 | VM의 네트워크 인터페이스 설정, VLAN 할당 |
| **VM-스토리지 연결** | 컴퓨트 + 스토리지 | VM 디스크 볼륨 연결, 스토리지 풀 매핑 |
| **스토리지 네트워크** | 네트워크 + 스토리지 | 스토리지 전용 VLAN, 고속 네트워크 설정 |
| **보안 정책 적용** | 네트워크 + 컴퓨트 | 테넌트별 보안 그룹, VM 방화벽 규칙 |
| **GPU VM 네트워크** | 컴퓨트 + 네트워크 | GPU VM의 AI 내부망/서비스망 연결 |
| **GPU VM 스토리지** | 컴퓨트 + 스토리지 | GPU VM의 학습 데이터 스토리지 연결 |
| **품질 게이트 스토리지** | 네트워크 + 스토리지 | 품질 게이트 → 공용 스토리지 데이터 흐름 |
| **모니터링 통합** | 전체 | Prometheus 메트릭 수집, Grafana 대시보드 |
| **IaC 통합** | 전체 | Terraform 모듈 통합, Ansible 플레이북 연계 |

### 4.3 AI 플랫폼 4계층별 담당

| 계층 | 담당 팀 | 역할 |
|------|--------|------|
| **계층 1: 테넌트 데이터 영역** | 컴퓨트 (데이터 업로드 VM), 네트워크 (VLAN), 스토리지 (Pool) | 테넌트 데이터 수집 환경 (업무 시스템 아님) |
| **계층 2: 품질 게이트** | 네트워크 (설계), 스토리지 (데이터 저장) | 데이터 검증 및 익명화 |
| **계층 3: 공용 AI 플랫폼** | 컴퓨트 (GPU), 스토리지 (학습 데이터) | GPU 클러스터, 학습 스토리지 |
| **계층 4: 서비스 계층** | 네트워크 (LB, API GW), 컴퓨트 (추론 서버) | 추론 서비스, 라우팅 |
