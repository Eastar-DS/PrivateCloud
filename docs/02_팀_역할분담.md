## 2. 팀별 역할 정의

### 2.1 컴퓨트/개발 팀 (2명)

#### 2.1.1 역할 개요

컴퓨트 계층의 설계 및 구현을 담당하며, 검증 워크로드 배포를 통해 인프라 동작을 확인한다.

#### 2.1.2 담당 영역

| 영역 | 세부 내용 |
|------|----------|
| **리소스 풀 설계** | 클러스터 풀 구성, 오버커밋 정책 |
| **VM 배치 전략** | Affinity/Anti-Affinity 규칙, 장애 도메인 분리 |
| **리소스 쿼터** | 테넌트별 vCPU/Memory 제한 정책 |
| **고가용성(HA)** | VM 자동 재시작, Live Migration 설정 |
| **VM 템플릿** | 표준 VM 이미지, 템플릿 관리 |
| **검증 워크로드** | 샘플 애플리케이션 배포, 인프라 동작 검증 |

#### 2.1.3 필요 역량

- KVM/QEMU 하이퍼바이저 이해
- 가상화 기술 (VM, 컨테이너) 기초
- Linux 시스템 관리
- 간단한 웹 애플리케이션 배포 능력

#### 2.1.4 학습 키워드

```
- KVM 하이퍼바이저
- 리소스 풀 (Resource Pool)
- Affinity / Anti-Affinity
- 장애 도메인 (Fault Domain)
- Live Migration
- VM 템플릿
- Cloud-init
```

---

### 2.2 네트워크/보안 팀 (2명)

#### 2.2.1 역할 개요

네트워크 계층의 설계 및 보안 아키텍처를 담당하며, 멀티테넌시 환경에서의 격리와 접근제어를 구현한다.

#### 2.2.2 담당 영역

| 영역 | 세부 내용 |
|------|----------|
| **망분리 설계** | 네트워크 존 정의, VLAN 할당 계획, IP 대역 설계 |
| **마이크로세그멘테이션** | 테넌트 간 격리, 서비스 간 통신 정책 |
| **방화벽 정책** | 계층별 접근 제어 규칙 정의 |
| **로드밸런서** | L4/L7 로드밸런싱 설정 |
| **RBAC 설계** | 역할 정의, 권한 매핑 |
| **암호화 체계** | TLS 설정, 저장 데이터 암호화 정책 |
| **감사 로깅** | 접근 로그, 감사 추적 |

#### 2.2.3 필요 역량

- 네트워크 기초 (OSI 7계층, TCP/IP)
- VLAN, 라우팅 이해
- 방화벽 개념 및 정책 설정
- 보안 기초 (인증, 인가, 암호화)

#### 2.2.4 학습 키워드

```
- VLAN / VXLAN
- SDN (Software Defined Networking)
- OVS (Open vSwitch)
- 마이크로세그멘테이션
- Zero Trust Security
- RBAC (Role-Based Access Control)
- TLS / PKI
- 방화벽 정책 (iptables, Security Group)
```

---

### 2.3 스토리지 팀 (2명)

#### 2.3.1 역할 개요

스토리지 계층의 설계 및 구현을 담당하며, 데이터의 가용성, 성능, 보호를 책임진다.

#### 2.3.2 담당 영역

| 영역 | 세부 내용 |
|------|----------|
| **분산 스토리지 설계** | Ceph 클러스터 구성, 복제 정책 |
| **티어링 정책** | Hot/Warm/Cold 정의, 자동 이동 규칙 |
| **테넌트 격리** | 스토리지 풀 분리, 쿼터 설정 |
| **스냅샷/백업** | 스냅샷 정책, 백업 자동화 |
| **DR 설계** | RPO/RTO 정의, 복제 전략 (설계 문서) |
| **성능 최적화** | 캐시 설정, I/O 튜닝 |

#### 2.3.3 필요 역량

- 스토리지 기초 (블록/파일/오브젝트)
- RAID 개념
- 분산 스토리지 이해
- 백업/복구 개념

#### 2.3.4 학습 키워드

```
- Ceph (분산 스토리지)
- OSD (Object Storage Daemon)
- MON (Monitor)
- 스토리지 풀 (Pool)
- 복제 (Replication)
- 티어링 (Tiering)
- 스냅샷 (Snapshot)
- DR (Disaster Recovery)
- RPO / RTO
```

## 4. 협업 포인트

### 4.1 교차 영역

일부 설계 영역은 여러 팀이 협업해야 합니다:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           협업 포인트                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────┐         ┌─────────────┐         ┌─────────────┐          │
│   │ 컴퓨트/개발 │◄───────►│네트워크/보안│◄───────►│  스토리지   │          │
│   └─────────────┘         └─────────────┘         └─────────────┘          │
│          │                       │                       │                  │
│          │                       │                       │                  │
│          ▼                       ▼                       ▼                  │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                        공통 협업 영역                                │  │
│   │                                                                     │  │
│   │  1. VM ↔ 네트워크 연결 (컴퓨트 + 네트워크)                          │  │
│   │  2. VM ↔ 스토리지 연결 (컴퓨트 + 스토리지)                          │  │
│   │  3. 스토리지 네트워크 (네트워크 + 스토리지)                          │  │
│   │  4. 모니터링 통합 (전체)                                            │  │
│   │  5. IaC 통합 (전체)                                                 │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 협업 상세

| 협업 영역 | 관련 팀 | 협업 내용 |
|----------|--------|----------|
| **VM-네트워크 연결** | 컴퓨트 + 네트워크 | VM의 네트워크 인터페이스 설정, VLAN 할당 |
| **VM-스토리지 연결** | 컴퓨트 + 스토리지 | VM 디스크 볼륨 연결, 스토리지 풀 매핑 |
| **스토리지 네트워크** | 네트워크 + 스토리지 | 스토리지 전용 VLAN, 고속 네트워크 설정 |
| **보안 정책 적용** | 네트워크 + 컴퓨트 | 테넌트별 보안 그룹, VM 방화벽 규칙 |
| **모니터링 통합** | 전체 | Prometheus 메트릭 수집, Grafana 대시보드 |
| **IaC 통합** | 전체 | Terraform 모듈 통합, Ansible 플레이북 연계 |

| 회의 | 주기 | 참석자 | 목적 |
